寄存器和数据存储

寄存器

首先认识 CPU 的构造

CPU 从逻辑上可以分为 3 个模块，分别是控制单元、运算单元和存储单元，这三部分由 CPU 内部总线连接起来。

而 寄存器 是一块速度非常快的计算机内存，下面是现代计算机中具有存储功能的部件比对，可以看到，寄存器的速度是最快的，同时也是造价最高昂的

寄存器是 CPU 内部的信息存储单元。它们是 CPU 中的一种高速存储器，用于临时存储指令和数据。寄存器的访问速度远快于外部内存（如 RAM），因此它们是实现高效指令处理的关键部分。

8086CPU 的寄存器

8086 共有 14 个寄存器，它们分别是:

  • 通用寄存器: AX, BX, CX, DX
  • 变址寄存器: SI、DI
  • 指针寄存器: SP、BP
  • 指令指针寄存器: IP
  • 段寄存器：CS、SS、DS、ES
  • 标志寄存器：PSW

对寄存器的掌握是 汇编语言的基础

通用寄存器

以 AX 为例：

AX 寄存器的位数是 16 位，每一位可以保存一个二进制位

如图：

这个存储器的最大值为 $2^{16}-1$，用十六进制表示为 FFFFH (H 是 16 进制标志位)

如果要存储一个十进制数，先将它转为二进制，然后让寄存器中对应的位由 0 变为 1 即可

 

在 8086 的上一代 CPU 中，寄存器只有 8 位。为了保证程序的兼容性，可以将 16 位通用寄存器分为两个 8 位寄存器使用，比如 将 16 位寄存器 AX 分为 AH 和 AL，将 BX 分为 BH 和 BL

8086 中“字”的存储

8086 是 16 位 CPU，所以 8086 的 字长 为 16bit

一个字可以存在一个 16 位的寄存器中：

  • 这个字的 高位字节 存在寄存器的 高 8 位寄存器
  • 这个字的 低位字节 存在寄存器的 低 8 位寄存器

例如通用寄存器 AX，高 8 位寄存器为 AH(High Byte)，低 8 位寄存器为 AL(Low Byte)

  • 如果你只需要操作整个寄存器，即 AX，则不需要考虑高低字节的存储
  • 如果要精确操作 AH 和 AL，则需要考虑字节的存储问题

例如， AX = 0x3412，那么高位 AH = 0x34，低位 AL = 0x12

CS，IP 寄存器

CS 和 IP 是 8086CPU 中两个最关键的寄存器，它们指示了 CPU 当前要读取的指令的地址

其中，CS 是代码段寄存器，IP 是指令指针寄存器

在 8086 机中，设 CS 的内容为 M，IP 内容为 N，8086CPU 从内存的M *16+N单元开始，读取指令并执行

也就是说，8086 机 在任意时刻，CPU 将 CS: IP 指向的内容当作指令执行

用 DS 和 [address] 实现字的传送

CPU 要读取一个内存单元的时候，必须先给出内存单元的地址

为了操作内存，我们使用 段存储器DS 存放 数据的段地址

CS, IP 是存储 指令 的地址，而 DS 存放的是 数据 的段的地址

示例汇编代码如下：

1
2
3

mov bx,1000H
mov ds,bx
mob al,[0]

这段代码的作用是：

  1. 设置数据段基址为 1000H。
  2. 从数据段内偏移地址 0（物理地址 10000H）处读取一个字节的数据到 AL。

注意：在 x86 汇编中，直接使用 MOV DS, 1000H 是 不合法的，因为在保护模式或实模式下，DS（数据段寄存器）这样的段寄存器不能直接赋值一个立即数。这是因为段寄存器必须通过寄存器间接加载。

8086CPU 可以一次性传送一个 字，例如：

3
4

mov bx,1000H
mov ds,bx
mov ax,[0] ;1000:0处的字型数据送入ax
mov [0],cx ;cx中的16位数据送到1000:0处

案例

注意：一个字(16 位)是两个字节(8 位)，一个存储单元只存储一个字节

所以 一个字占用了两个存储单元

字的存储按照 小端序，即 高位地址存高位，低位地址存低位

DS 和数据段

8086 机可以根据需要将一组内存单元定义为一个段

  • 物理地址 = 段地址 ×16+偏移地址
  • 将一组长度位 N(N <= 64K)、地址连续、起始地址为 16 的倍数 的内存单元当作专门存储数据的内存空间，从而定义了一个数据段

8086 寄存器中有两个与栈相关的寄存器：

  • 栈顶寄存器 SS -存放 栈顶的段地址
  • 栈顶指针寄存器 SP -存放 栈顶的偏移地址

SS 和 SP 用于确定被用作栈的内存的栈顶地址

ss 和 sp 的使用

示例代码如下：

mov ax,1000H
mov ss,ax
mov sp,0010H

mov ax,001AH
mov bx,001BH

push ax
push bx

pop ax
pop bx

代码分析：

  1. 初始化栈段和栈指针，设置栈的起始位置。栈段基址是 1000H，栈指针为 0010H。
  2. 将两个值（26 和 27）分别存入 AX 和 BX 寄存器，准备后续操作(小端序)
  3. 将寄存器 ax, bx 中的值压入栈
  4. 将栈顶的数据分别弹回到寄存器 ax 和 bx

该代码实现了一个 寄存器 ax 和 bx 的数据交换，意义等同于 c 语言中交换变量值的操作

在汇编语言中，8086CPU 不会保证对栈的操作不会越界，因为 8086CPU 只知道栈顶的位置，不知道程序安排的栈空间是多大

栈顶超界的问题需要我们自己操心，要根据可能用到的最大栈空间，安排栈的大小
