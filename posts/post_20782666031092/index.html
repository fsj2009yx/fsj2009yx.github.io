算法基础

事前分析法和事后分析法

事前分析法通常包括时间复杂度和空间复杂度的计算，利用大O表示法来预测算法在最坏、最好和平均情况下的表现。这种方法有助于在选择数据结构和算法时做出明智的决策，并可以揭示潜在的性能瓶颈。

事后分析法则侧重于通过实验和实际运行数据来评估算法的性能。通过收集运行时间、内存使用等指标，开发者可以发现与理论分析不符的地方，进行调整和优化。这种方法在面对复杂或动态环境时尤其重要，因为理论分析可能无法完全捕捉实际情况。

时间复杂度和空间复杂度的计算都属于事前分析法

线性表

单向链表

单向链表（Singly Linked List）是一种线性数据结构，其中每个节点包含一个数据域和一个指向下一个节点的指针域

基本结构

  • 节点：每个节点由两部分组成：
    1. 数据域：存放数据元素。
    2. 指针域：存放指向下一个节点的指针。
  • 尾节点：最后一个节点的指针域指向null，表示链表的结束

基本类型

带头节点的单向链表：

头节点是链表的第一个节点，但不存储实际数据，只作为链表的入口

结构如下：

1

[head] -> [data1] -> [data2] -> [data3] -> null

头指针：

在带头节点的链表中，头指针指向的是头节点，而头节点通常不存放数据，仅作为链表的入口，帮助简化链表操作。

  • 含义：头指针是指向头节点的指针，头节点的下一个节点才是存放数据的第一个实际节点。头指针始终存在，并且在链表初始化时指向头节点，即使链表中没有数据节点，头指针也不会为 null。
  • 优点：带头节点的链表使得插入和删除操作，尤其是在链表头部进行的操作更为简便，因为可以统一从头节点后操作，不需要专门处理特殊情况。

在单向链表中，头指针是一个指针变量，指向链表的第一个节点。它在链表的管理中起着至关重要的作用。头指针的含义在带头节点和无头节点的链表中有所不同。

不带头节点的单向链表：

没有专门的头节点，链表的第一个节点直接存放数据。指向链表的入口是第一个数据节点。

结构如下：

1

[data1] -> [data2] -> [data3] -> null

头指针：

在无头节点的链表中，头指针指向的是链表中的第一个数据节点，没有专门的头节点来辅助操作。

  • 含义：头指针直接指向链表的第一个实际存放数据的节点。如果链表为空，头指针会指向 null。
  • 优点：无头节点的链表结构简单，节省了一个节点的空间。但在执行插入或删除第一个节点时，需要特殊处理。

class Stack {
private:
    T* elements;      // 存储栈元素的数组
    int capacity;     // 栈的最大容量
    int top;          // 栈顶指针
    int base;         // 栈底指针

public:
    Stack(int size) : capacity(size), top(0), base(0) {
        elements = new T[capacity]; // 动态分配数组
    }

    ~Stack() {
        delete[] elements; // 释放动态分配的内存
    }

    void push(const T& value) {
        if (top >= capacity) {
            throw std::overflow_error("Stack overflow!"); // 栈满时抛出异常
        }
        elements[top++] = value; // 存储元素并增加栈顶指针
    }

    void pop() {
        if (isEmpty()) {
            throw std::underflow_error("Stack underflow!"); // 栈空时抛出异常
        }
        --top; // 移除栈顶元素
    }

    T topElement() const {
        if (isEmpty()) {
            throw std::out_of_range("Stack is empty!"); // 栈空时抛出异常
        }
        return elements[top - 1]; // 返回栈顶元素
    }

    bool isEmpty() const {
        return top == base; // 判断栈是否为空
    }

    bool isFull() const {
        return top >= capacity; // 判断栈是否满
    }

    size_t size() const {
        return top - base; // 返回当前栈的大小
    }
};

int main() {
    Stack<int> intStack(5); // 创建容量为5的栈

    intStack.push(1);
    intStack.push(2);
    intStack.push(3);
    std::cout << "Top element: " << intStack.topElement() << "\n";
    intStack.pop();
    std::cout << "Top element after pop: " << intStack.topElement() << "\n";

    return 0;
}

#include <iostream>
#include <stdexcept>

// 链栈节点
template <typename T>
struct Node {
    T data;          // 数据域
    Node* next;     // 指向下一个节点的指针

    Node(const T& value) : data(value), next(nullptr) {}
};

// 链栈类
template <typename T>
class LinkedStack {
private:
    Node<T>* topNode; // 栈顶节点

public:
    LinkedStack() : topNode(nullptr) {} // 初始化栈为空

    ~LinkedStack() {
        while (!isEmpty()) {
            pop(); // 释放所有节点
        }
    }

    void push(const T& value) {
        Node<T>* newNode = new Node<T>(value); // 创建新节点
        newNode->next = topNode;               // 新节点指向当前栈顶
        topNode = newNode;                     // 更新栈顶指针
    }

    void pop() {
        if (isEmpty()) {
            throw std::underflow_error("Stack underflow!"); // 栈空时抛出异常
        }
        Node<T>* temp = topNode;    // 记录当前栈顶节点
        topNode = topNode->next;     // 更新栈顶指针
        delete temp;                // 释放旧的栈顶节点
    }

    T top() const {
        if (isEmpty()) {
            throw std::out_of_range("Stack is empty!"); // 栈空时抛出异常
        }
        return topNode->data;      // 返回栈顶节点的数据
    }

    bool isEmpty() const {
        return topNode == nullptr; // 判断栈是否为空
    }

    size_t size() const {
        size_t count = 0;
        Node<T>* current = topNode;
        while (current) {
            count++;
            current = current->next;
        }
        return count; // 返回栈中节点的数量
    }
};

int main() {
    LinkedStack<int> intStack; // 创建链栈

    intStack.push(1);
    intStack.push(2);
    intStack.push(3);
    std::cout << "Top element: " << intStack.top() << "\n";
    intStack.pop();
    std::cout << "Top element after pop: " << intStack.top() << "\n";

    return 0;
}

#include <iostream>
#include <stdexcept>

using namespace std;

template <typename T>
class ArrayQueue 
{
private:
    T* data;         // 存储队列元素的数组
    int front;       // 队头索引
    int rear;        // 队尾索引
    int capacity;    // 队列容量
    int size;        // 当前队列大小

public:
    //初始化队列
    ArrayQueue(int cap) : capacity(cap), front(0), rear(-1), size(0) 
    {
        data = new T[capacity];
    }

    ~ArrayQueue()
    {
        delete[] data;
    }

    // 入队操作
    void enqueue(const T& item) 
    {
        if (size == capacity) 
        {
            throw overflow_error("Queue is full");
        }
        rear = (rear + 1) % capacity;//rear指向下一个元素
        data[rear] = item;//将元素放入队尾
        size++;
    }

    // 出队操作
    void dequeue() 
    {
        if (isEmpty()) 
        {
            throw underflow_error("Queue is empty");
        }
        front = (front + 1) % capacity;//front指向下一个元素
        size--;
    }

    // 获取队头元素
    T frontElement() const 
    {
        if (isEmpty()) 
        {
            throw underflow_error("Queue is empty");
        }
        return data[front];
    }

    // 判断队列是否为空
    bool isEmpty() const 
    {
        return size == 0;
    }

    // 获取队列大小
    int getSize() const
    {
        return size;
    }

    // 清空队列
    void clear() 
    {
        front = 0;
        rear = -1;
        size = 0;
    }

    // 显示队列元素
    void display() const
    {
        if (isEmpty())
        {
            cout << "Queue is empty" << endl;
            return;
        }
        cout << "Queue elements: ";
        for (int i = 0; i < size; i++)
        {
            cout << data[(front + i) % capacity] << " ";
        }
        cout << endl;
    }
};

int main() 
{
    ArrayQueue<int> queue(5);

    queue.enqueue(1);
    queue.enqueue(2);
    queue.enqueue(3);

    queue.display();

    std::cout << "Front element: " << queue.frontElement() << std::endl;

    queue.dequeue();
    std::cout << "Front element after dequeue: " << queue.frontElement() << std::endl;

    queue.clear();
    queue.display();

    return 0;
}

假上溢

队列的假上溢（False Overflow）是指在队列中，队列看似已满，但实际上仍有空间可以存放新元素。这种情况通常发生在使用固定大小的数组实现队列时。为了避免这种情况，通常会使用一个额外的变量来跟踪队列的长度，或使用一种特殊的标记来区分队列是空还是满。

举例说明

假设我们有一个容量为 5 的顺序队列，用数组实现：

  • 队列初始状态：front = 0, rear = 0，数组为 [_, _, _, _, _]（_ 表示空位）。

我们进行如下操作：

  1. 入队操作：依次插入 5 个元素。
    • 插入 1：[1, _, _, _, _]，front = 0, rear = 1。
    • 插入 2：[1, 2, _, _, _]，front = 0, rear = 2。
    • 插入 3：[1, 2, 3, _, _]，front = 0, rear = 3。
    • 插入 4：[1, 2, 3, 4, _]，front = 0, rear = 4。
    • 插入 5：[1, 2, 3, 4, 5]，front = 0, rear = 5。
  2. 出队操作：移除前 3 个元素。
    • 移除 1：[_, 2, 3, 4, 5]，front = 1, rear = 5。
    • 移除 2：[_, _, 3, 4, 5]，front = 2, rear = 5。
    • 移除 3：[_, _, _, 4, 5]，front = 3, rear = 5。

此时，数组仍有空间，但由于 rear 达到了数组末尾，再插入新元素时会认为队列已满，这是因为没有复用出队腾出的空间。

关键现象
  1. 数组中仍有空位（例如 0, 1, 2 索引位置为空）。
  2. 队列逻辑却认为已经“满了”，导致无法继续入队。
解决假上溢的方法

  1. 遍历移动front和rear元素向数组首部方向，缺点是浪费时间

  2. 使用循环队列解决

#include <iostream>
#include <stdexcept>

template <typename T>
class CircularQueue {
private:
    T* data;           // 存储队列元素的数组
    int front;         // 队列头指针
    int rear;          // 队列尾指针
    int capacity;      // 队列容量
    int size;          // 当前队列大小

public:
    CircularQueue(int cap) : capacity(cap), front(0), rear(0), size(0)
    {
        data = new T[capacity];
    }

    ~CircularQueue() 
    {
        delete[] data;
    }

    bool isFull() const 
    {
        return size == capacity-1;//容量为5，size为4时，队列已满，因为front和rear指向同一个位置时，队列是空的
        //留空一个元素空间的目的是为了区分队列满和队列空的情况
    }

    bool isEmpty() const 
    {
        return size == 0;
    }

    void enqueue(const T& value)
    {
        if (isFull()) 
        {
            throw std::overflow_error("Queue is full");
        }
        data[rear] = value;
        rear = (rear + 1) % capacity;  // 环绕，当rear到达capacity-1时，回到0
        size++;
    }

    T dequeue()
    {
        if (isEmpty()) 
        {
            throw std::underflow_error("Queue is empty");
        }
        T value = data[front];
        front = (front + 1) % capacity; // 环绕，当front到达capacity-1时，回到0
        size--;
        return value;
    }

    T peek() const 
    {
        if (isEmpty()) 
        {
            throw std::underflow_error("Queue is empty");
        }
        return data[front];
    }

    int getSize() const 
    {
        return size;
    }

    int getCapacity() const 
    {
        return capacity;
    }
};

int main() {
    CircularQueue<int> queue(5);

    try {
        queue.enqueue(1);
        queue.enqueue(2);
        queue.enqueue(3);
        queue.enqueue(4);
        queue.enqueue(5);

        std::cout << "Front element: " << queue.peek() << std::endl;

        while (!queue.isEmpty()) 
        {
            std::cout << "Dequeued: " << queue.dequeue() << std::endl;
        }

        // Demonstrate wraparound
        queue.enqueue(6);
        queue.enqueue(7);
        std::cout << "Front element after wraparound: " << queue.peek() << std::endl;

    }
    catch (const std::exception& e) {
        std::cerr << e.what() << std::endl;
    }

    return 0;
}

双端队列

双端队列（Deque, double-ended queue）是一种数据结构，它具有以下性质：

  1. 双端操作：双端队列允许在两端（前端和后端）进行插入和删除操作，既可以从队列头部插入或删除元素，也可以从队列尾部插入或删除元素。
  2. 灵活性：由于双端队列支持双端操作，它可以用作队列（FIFO）和栈（LIFO）。当只在一端插入和删除时，双端队列可以模拟栈的行为，而当分别从头部移出和尾部插入时，则可以模拟队列的行为。
  3. 有序性：双端队列中元素保持顺序，插入顺序决定了元素的排列。
  4. 时间复杂度：在双端队列的前端和后端进行插入和删除操作的时间复杂度通常是 O(1)。

双端队列的应用：

尽管队列和栈已经存在，双端队列提供了更多的灵活性和功能，可以适应一些队列和栈无法高效处理的场景。以下是双端队列相比普通队列和栈的优势：

1. 双端操作的灵活性：

双端队列支持在两端进行插入和删除操作，这使得它能够灵活地充当：

  • 队列：从一端进，从另一端出（FIFO）。
  • 栈：只在一端插入和删除（LIFO）。
  • 双向队列：可以在两端自由插入和删除数据，这是普通队列和栈无法实现的。

2. 更高效的特殊操作：

在一些应用场景中，双端队列的灵活性可以带来显著的性能提升。例如：

  • 滑动窗口问题：在一些算法中，如滑动窗口最大值或最小值问题，双端队列可以高效地从一端插入新元素，同时从另一端删除不再需要的元素。这比单纯使用栈或队列要高效得多。
  • 缓存机制：双端队列常用于LRU（最近最少使用）缓存的实现，在前端和后端都可以高效操作，方便管理缓存。

3. 双向遍历或搜索：

在某些情况下，需要从两端同时遍历或处理数据，例如双向搜索算法（如双向广度优先搜索），使用双端队列可以简化代码并提高性能。
